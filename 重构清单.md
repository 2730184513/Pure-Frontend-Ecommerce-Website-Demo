## 1.1 `product-repository.js` (核心重构)

**当前问题**：

- 依赖外部的 `product-data-loader.js`
- 包含 Load More 按钮逻辑
- 包含商品卡渲染逻辑
- 非单例模式

**重构要求**：

1. **改为全局单例模式**，与 `cartManager` 同级使用
	- 在 `window` 对象上暴露：`window.productRepository`
2. **职责定义**：
	- ✅ **自行加载全量 Product 数据**（不再依赖 `product-data-loader.js`）
	- ✅ 加载单个 JSON 文件（全量数据，例如 `products.json`）
	- ✅ 提供 CRUD 操作：
		- `getAll()` - 返回所有商品副本
		- `getById(id)` - 根据 ID 查找单个商品
		- `add(product)` - 添加商品（未来扩展）
		- `update(id, product)` - 更新商品（未来扩展）
		- `delete(id)` - 删除商品（未来扩展）
	- ✅ 提供查询过滤功能：
		- `query(filterFn)` - 接受过滤函数参数，返回过滤后的商品数组
		- 参数 `filterFn` 格式：`(product) => boolean`
	- ✅ 提供排序功能：
		- `sort(products, sortFn)` - 接受商品数组和排序函数，返回排序后的数组
		- 参数 `sortFn` 格式：`(a, b) => number`
	- ❌ **移除**：Load More 按钮逻辑 → 迁移到 `home-manager.js`
	- ❌ **移除**：商品卡渲染逻辑 → 由 `product-card-renderer.js` 统一负责
	- ❌ **移除**：基于 Category 加载特定 JSON 文件的功能（此功能在 `product-data-loader.js` 中）
3. **数据加载说明**：
	- Repository 内部直接使用 `fetch` 或 `XMLHttpRequest` 加载 JSON 数据
	- 加载完成后存储在实例的私有属性中（如 `this.products`）
	- 提供 `async loadAll()` 方法供外部调用初始化
4. **查询过滤说明**：
	- `query(filterFn)` 方法对内部存储的 `this.products` 执行 `Array.filter(filterFn)`
	- 返回过滤后的商品数组副本（防止外部修改原数组）
5. **排序说明**：
	- `sort(products, sortFn)` 方法对传入的商品数组执行 `Array.sort(sortFn)`
	- 返回排序后的新数组（不修改原数组）

**文件位置**：`\201-project\js\core\product-repository.js`

## 🟠 任务 2: 业务逻辑层重构 - Filter 单例化

## 2.1 `product-filter.js` (核心重构)

**当前问题**：

- 非单例模式
- 缺少 SessionStorage 持久化
- 缺少完整的 getter/setter
- 字段定义不明确

**重构要求**：

1. **改为全局单例模式**（类似 `cartManager`）：

	- 在 `window` 对象上暴露：`window.productFilter`

2. **字段定义**（共 11 个字段）：

	**需要计数的字段（7 个）**：

	- `categories` (数组) - 选中的类别列表（每个类别独立计数）
	- `minPrice` (数值) - 最低价格
	- `maxPrice` (数值) - 最高价格
	- `minRate` (数值) - 最低评分
	- `maxRate` (数值) - 最高评分
	- `from` (字符串/日期) - 起始日期
	- `to` (字符串/日期) - 结束日期

	**不计数的字段（4 个）**：

	- `searchKeyword` (字符串) - 搜索关键词
	- `itemsPerPage` (数值) - 单页显示数量
	- `sortKey` (字符串) - 排序字段（如 'price', 'rating', 'name'）
	- `sortOrder` (字符串) - 排序方向（'asc' / 'desc' / 'default'）

3. **职责定义**：

	- ✅ **存储所有过滤条件**（上述 11 个字段）
	- ✅ **提供 getter/setter**（支持链式调用）：
		- 每个字段提供对应的 `get` 和 `set` 方法
		- `set` 方法在修改字段后自动调用 `saveToStorage()`
		- `set` 方法返回 `this` 支持链式调用
	- ✅ **提供字段检测逻辑**（供 Repository 使用）：
		- `match(product)` - 判断单个 product 是否匹配所有过滤条件
		- `toFilterFunction()` - 返回过滤函数，格式为 `(product) => this.match(product)`
	- ✅ **提供 filter count 计算**（仅计数 7 个需要计数的字段）：
		- `getActiveFilterCount()` - 返回激活的过滤条件数量
		- 计数规则：
			- 如果 `categories` 数组有 3 个元素，计数为 3
			- 如果 `minPrice` 不等于默认值（如 0），计数 +1
			- 如果 `maxPrice` 不等于默认值（如 10000），计数 +1
			- 其他字段类似逻辑
			- **不计数**：`searchKeyword`, `itemsPerPage`, `sortKey`, `sortOrder`
	- ✅ **SessionStorage 持久化**：
		- `saveToStorage()` - 自动保存到 `sessionStorage`
		- `loadFromStorage()` - 从 `sessionStorage` 恢复（在构造函数中调用）
		- `clearStorage()` - 清除持久化数据
	- ✅ **重置方法**：
		- `reset()` - 重置所有字段为默认值（调用后自动保存到 Storage）

4. **默认值定义**：

	- `categories: []`
	- `minPrice: 0`
	- `maxPrice: 10000`
	- `minRate: 0`
	- `maxRate: 5`
	- `from: null`
	- `to: null`
	- `searchKeyword: ''`
	- `itemsPerPage: 16`
	- `sortKey: 'default'`
	- `sortOrder: null`

5. **filter count 计算示例**：

	- 选中 3 个 categories → count = 3
	- minPrice = 100 → count +1
	- maxPrice = 5000 → count +1
	- minRate = 4 → count +1
	- maxRate = 5 → 不计数（等于默认值）
	- from = '2024-01-01' → count +1
	- to = null → 不计数（等于默认值）
	- searchKeyword = 'sofa' → **不计数**
	- 总计：count = 7

**文件位置**：`\201-project\js\pages\shop\product-filter.js`
