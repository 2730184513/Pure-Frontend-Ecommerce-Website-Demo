## 1. cart.js - 购物车管理器

### 核心功能
购物车的**业务逻辑层**，负责购物车数据的管理、存储和状态维护。

### 主要类: `CartManager`

#### 职责范围
- ✅ 购物车数据管理（增删改查）
- ✅ localStorage 持久化存储
- ✅ 购物车状态同步
- ✅ 全局事件监听和分发
- ✅ 通知消息管理

#### 核心方法

| 方法名 | 参数 | 功能描述 |
|--------|------|----------|
| `init()` | 无 | 初始化购物车，加载数据，设置事件监听器，初始化下拉渲染器 |
| `loadCart()` | 无 | 从 localStorage 加载购物车数据 |
| `saveCart()` | 无 | 保存购物车数据到 localStorage |
| `addProduct(product)` | `product`: 商品对象 | 添加商品到购物车，自动处理变体 ID 和数量 |
| `updateQuantity(variantId, delta)` | `variantId`: 变体 ID<br>`delta`: 数量变化（+1/-1） | 更新商品数量，确保最小数量为 1 |
| `removeProduct(variantId, silent)` | `variantId`: 变体 ID<br>`silent`: 是否静默删除 | 从购物车移除商品 |
| `getCart()` | 无 | 获取购物车商品列表副本 |
| `getCount()` | 无 | 获取购物车中独特变体的数量 |
| `showNotification(message, type)` | `message`: 消息内容<br>`type`: 通知类型 | 显示 Toast 通知 |

#### 数据结构

**购物车商品对象 (Cart Item)**
```javascript
{
    id: "chair-0",                    // 商品基础 ID
    variantId: "chair-0-L-#FFFFFF",  // 变体 ID（id-尺寸-颜色）
    name: "Product Name",             // 商品名称
    price: 1299,                      // 价格
    product_picture: "/path/to/img",  // 商品图片
    selectedSize: "L",                // 选中的尺寸
    selectedColor: "#FFFFFF",         // 选中的颜色
    qty: 2,                           // 数量
    // ...其他商品属性
}
```

#### 事件交互

**监听的事件**
- `addToCart` - 全局添加到购物车事件
  - 触发源: `ProductPopup` (pop-up.js)
  - 数据: `{product: 商品对象}`

**分发的事件**
- `cartUpdated` - 购物车更新事件
  - 触发时机: 添加、更新、删除商品后
  - 监听者: 购物车页面组件

#### 依赖关系

**依赖的模块**
- `CartDropdownRenderer` - 购物车下拉渲染器（在 init() 中创建实例）
- `window.toast` (ToastManager) - 通知系统

**被依赖的模块**
- 购物车页面 (cart.html)
- 结账页面 (checkout.html)

#### 存储机制
- **存储键**: `furniro_cart`
- **存储位置**: `localStorage`
- **数据格式**: JSON 序列化的商品数组

## 2. cart-dropdown-renderer.js - 购物车下拉渲染器

### 核心功能
购物车的**视图层**，负责头部购物车下拉框的渲染和交互。

### 主要类: `CartDropdownRenderer`

#### 职责范围
- ✅ 购物车下拉框的 DOM 渲染
- ✅ 购物车徽章显示和更新
- ✅ 用户交互处理（悬停、点击、数量调整）
- ✅ 图片懒加载
- ✅ 购物车图标动画

#### 核心方法

| 方法名 | 参数 | 功能描述 |
|--------|------|----------|
| `init()` | 无 | 初始化渲染器，创建 DOM 结构，绑定事件 |
| `cacheDOM()` | 无 | 缓存常用 DOM 元素 |
| `createDropdown()` | 无 | 动态创建下拉框容器 |
| `createBadge()` | 无 | 创建购物车徽章元素 |
| `bindEvents()` | 无 | 绑定所有事件监听器（点击、悬停、双击） |
| `showDropdown()` | 无 | 显示下拉框，渲染购物车内容 |
| `updateBadge()` | 无 | 更新徽章显示的商品数量 |
| `playIconAnimation()` | 无 | 播放购物车图标添加动画 |
| `render()` | 无 | 渲染购物车商品列表 |
| `createCartItemElement(item)` | `item`: 购物车商品 | 创建单个购物车商品的 DOM 元素 |
| `lazyLoadImages(container)` | `container`: 容器元素 | 懒加载商品图片 |
| `handleIncreaseQuantity(variantId)` | `variantId`: 变体 ID | 处理增加数量 |
| `handleDecreaseQuantity(variantId)` | `variantId`: 变体 ID | 处理减少数量 |
| `handleDeleteItem(variantId)` | `variantId`: 变体 ID | 处理删除商品 |
| `handleQuantityInputChange(input)` | `input`: 输入框元素 | 处理数量输入框变化 |

#### 交互特性

**悬停行为**
- 悬停延迟: 500ms
- 悬停图标或下拉框时保持显示
- 鼠标离开时自动关闭

**点击行为**
- 单击图标: 立即显示下拉框
- 双击图标: 跳转到购物车页面
- 点击商品区域: 阻止冒泡

**数量控制**
- 最小数量: 1
- 最大数量: 9999
- 输入验证: 自动修正无效输入

#### 图片懒加载
```javascript
// 使用占位图 + data-src 属性
<img src="/placeholder.jpg" data-src="/actual-image.jpg">
```
- 初始显示占位图
- 渲染后异步加载实际图片
- 加载失败保留占位图

#### 徽章显示规则
- 数量 = 0: 隐藏徽章
- 数量 1-99: 显示实际数量
- 数量 > 99: 显示 "99+"

#### 依赖关系

**依赖的模块**
- `CartManager` - 购物车管理器（构造函数注入）
- `window.toast` (ToastManager) - 通知系统

**被依赖的模块**
- `CartManager` - 在其 init() 中创建实例

## 5. search.js - 搜索管理器

### 核心功能
全局搜索功能的管理，负责搜索浮层的显示和搜索操作的处理。

### 主要类: `SearchManager`

#### 职责范围
- ✅ 搜索浮层的显示和隐藏
- ✅ 搜索输入框管理
- ✅ 搜索跳转逻辑
- ✅ 键盘事件处理

#### 核心方法

| 方法名 | 参数 | 功能描述 |
|--------|------|----------|
| `init()` | 无 | 初始化搜索管理器，缓存 DOM，绑定事件 |
| `cacheDOM()` | 无 | 缓存常用 DOM 元素 |
| `bindEvents()` | 无 | 绑定所有事件监听器 |
| `openSearch()` | 无 | 打开搜索浮层，自动聚焦输入框 |
| `closeSearch()` | 无 | 关闭搜索浮层，清空输入框 |
| `performSearch()` | 无 | 执行搜索操作，跳转到商店页面 |

#### DOM 结构依赖

需要以下 DOM 元素存在:
- `#icon-search` - 搜索图标
- `#search-overlay` - 搜索浮层
- `#global-search-input` - 搜索输入框
- `#global-search-btn` - 搜索按钮

#### 搜索流程

1. **打开搜索框**
   - 点击搜索图标
   - 移除 `hidden` 类
   - 100ms 后自动聚焦输入框

2. **执行搜索**
   - 点击搜索按钮 或 按 Enter 键
   - 获取输入内容（trim）
   - 保存到 `localStorage` (`shop_search_query`)
   - 跳转到 `shop.html`

3. **关闭搜索框**
   - 点击浮层背景
   - 添加 `hidden` 类
   - 清空输入框

#### 存储机制
- **存储键**: `shop_search_query`
- **存储位置**: `localStorage`
- **数据格式**: 字符串（搜索关键词）

#### 特殊处理
- 允许空字符串搜索（表示显示所有商品）
- 自动 trim 输入内容
- 跳转到商店页面时由商店页面读取搜索关键词

#### 依赖关系

**依赖的模块**
- 无直接依赖

**被依赖的模块**
- 商店页面 (shop.html) - 读取 `shop_search_query` 并执行搜索

## 架构模式总结

### 1. **业务逻辑与视图分离**
- **Manager 类**: 负责数据和业务逻辑
- **Renderer 类**: 负责 DOM 渲染和用户交互
- 优势: 高内聚、低耦合、易测试、易维护

### 2. **事件驱动架构**
- 使用 `CustomEvent` 进行组件间通信
- 松耦合的组件协作
- 易于扩展和维护

### 3. **单例模式**
- 所有管理器在全局作用域只有一个实例
- 通过 `isInitialized` 标志防止重复初始化

### 4. **依赖注入**
- Renderer 类通过构造函数接收 Manager 实例
- 提高可测试性和灵活性

### 5. **数据持久化**
- 使用 `localStorage` 存储用户数据
- JSON 序列化和反序列化
- 数据与视图解耦

## 性能优化策略

### 1. **图片懒加载**
- 使用占位图提升首次渲染速度
- 异步加载实际图片
- 错误处理：保留占位图

### 2. **DOM 缓存**
- `cacheDOM()` 方法缓存常用元素
- 减少重复查询

### 3. **事件委托**
- 在容器级别绑定事件
- 减少事件监听器数量

### 4. **防止重复初始化**
- `isInitialized` 标志
- 单例模式确保唯一实例
