This project is merely a school course assignment outcome, a pure front-end furniture e-commerce website simulated based on HTML+JS+CSS. For buyers, it realizes functions such as ***account registration, login, search, pagination, filtering, product detail page, model selection, wish list, shopping cart, and checkout***. For sellers, they can ***use the administrator account to log in and manage existing products as well as add new products.*** Additionally, this project is compatible with both desktop and mobile versions. Although the layout and interaction logic may be incomplete when accessed on mobile devices—there might be issues on certain screen sizes.

Since the image resources are not stored in an external OSS but are instead stored in the form of files in the "images" folder, loading them for the first time without caching can be rather slow. 

User data and transaction data are cached in the browser in the form of sessions. Therefore, using different browsers or accessing across platforms will result in the loss of previous operation history and the restoration to the initial state of the project.
